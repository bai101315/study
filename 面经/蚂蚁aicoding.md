PARSE 法则

# Plan 规划（0-10分钟）

```
我需要在2小时内完成以下项目：

[把题目README完整贴这里，一个字都不要删]

请先不要写代码，帮我分析：
1. 技术选型建议（给2-3个方案，标注推荐）
2. 模块划分和文件结构
3. 核心数据模型设计
4. 接口/API设计
5. 实现优先级排序（哪些必须做，哪些是加分项）

用简洁格式输出，我需要快速决策。

推荐选型直接用，不要纠结：Python + FastAPI + rich + HTTP轮询 + 内存dict（可选SQLite）+ pytest。

```

# A - Architect 架构实现（10-40分钟）
```
技术栈确认：[你选的技术栈]

请实现 [核心文件名]，包含：
1. [数据模型要求]
2. [业务逻辑要求]
3. [API/接口要求]

验收标准：[写清楚怎么算"好了"]
输出完整可运行代码。
```

# R - Realize 前端/界面（40-65分钟）

```
后端API已就绪（[地址]），现在实现 [模块名]：
- [功能点1]
- [功能点2]
- [功能点3]
- 输入非法内容不崩溃

验收标准：[怎么算完成]
```

# S - Stabilize 稳定化（65-90分钟）

```
项目基本功能完成，请创建 [测试文件名]（用[测试框架]），覆盖：
- [正向流程测试]
- [异常输入测试]
- [权限/边界测试]

要求：
- 测试自包含，不需要手动启动服务
- 每个测试独立，不依赖执行顺序
- 附上运行命令

同时生成 run_tests.sh：一键运行所有测试并保存报告。
```

# E - Export 交付（90-120分钟）
```
基于项目实际代码生成使用说明文档，包含：
- 环境要求和安装步骤
- 启动方式
- 使用流程（每步操作和预期结果）
- 项目文件结构和各文件职责
- 设计决策说明（为什么选这个技术方案）

不要编造不存在的功能。
```
