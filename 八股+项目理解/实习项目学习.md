# 字符串清洗
## 1，主副标题拆分
存在特殊符号、标题中存在  “第*季” 、标题中存在单独的数字 存在以上情况就会作为子标题(sub-title)

原始标题里经常混有冒号、书名分隔点、方括号、季数、尾部数字等信息。这些内容如果直接送去排版，系统只会把它们当成普通字符，导致主标题过长、排版拥挤、视觉重点不清。所以第一步先把“主标题”和“附属信息”用空格切开。

这里的设计重点是：第一段永远作为视觉主标题，后面的内容合并成副标题，保证主视觉不会被过长文本拖垮。


## 判断是否折行，缩放比例、错落幅度

业务那边强制要求，分为现代与非现代
现代：>=7, 强制折行
非现代：>= 6, 强制折行

输入：
```
title: 主标题
subtitle: 副标题
style: 风格，比如现代、古装、悬疑犯罪、年代抗战、少儿
image_vert: 图片是否竖版
folder_pos: 不同横竖版下的折行阈值
nline: 用户是否强制指定单行或双行
```

输出：
```
scale_rito: 字体错落/缩放相关比例
offset_rito: 字符偏移幅度比例
sent_cut: 是否需要把主标题切成两行
```

设计理念：
```
不同风格的标题审美不一样。
现代剧标题常常更适合横向、轻微错落，比如："你给我的喜欢", "爱的二八定律"

古装、悬疑、年代抗战等风格，对大字标题、错落排版、折行的接受度更高。
```


缩放与偏移比例：
```
scale_rito, offset_rito = 0.15, 0.1
scale_rito, offset_rito = 0.15, 0.25
```

## 如果需要断行的话

设计理念
```
断句不能简单从中间砍。中文标题里有词语、助词、连接词、语素，如果拆错会很怪。
比如："知否知否应是绿肥红瘦"
不能机械切成："知否知否应" / "是绿肥红瘦"
```

具体操作：
```
第一步，用 jieba.posseg 做分词和词性标注
第二步，收集候选断点:前半部分长度 >= 2; 后半部分长度 >= 2。 这样避免出现：这种头重脚轻或尾重头轻的结果。

第三步，记录特殊词性位置：
connector_positions: 介词、连词位置
single_char_positions: 单字词位置
g_char_positions: 语素位置
particle_char_positions: 助词位置

第四步，对每个候选断点打分。
主要看几个因素：
长度是否均衡、是否拆开相邻单字、是否把语素单独拆开、是否把助词和前面的词拆开、是否靠近中间

```

## 判断副标题排版方式
```vert_subtitle, hori_subtitle, mid_subtitle = get_subtitle_type(subtitle, nline)```

输入：
subtitle: 副标题字符串
nline: 用户指定行数

输出：
```
vert_subtitle: 副标题是否竖排
hori_subtitle: 副标题是否横向拼接
mid_subtitle: 副标题是否居中拼接
```

设计理念：
```
副标题通常比主标题小，主要承担补充信息，不应该抢主标题的视觉中心。因此系统会根据副标题长度和字符类型决定它贴在主标题旁边、下面，还是居中。
```

具体操作：
```
先统计副标题里中文字符数量：
判断是否纯数字或英文：
如果副标题是 3 个以内的纯中文：
可能竖排，也可能横排。没有指定 nline 时，会带一点随机性，让生成结果更多样。

如果 nline == 1：倾向竖排副标题
如果 nline == 2：倾向横排副标题
```

## 为后续字体排版准备字符级信息

```words = get_font_rito(title, "0", font_vert, font_path, scale_rito, mode)```

输入：
```
title: 标题字符串
font_vert: 是否竖排
font_path: 当前字体
scale_rito: 错落/缩放比例
mode: 布局模式
```

输出：
```
[(char, ratio), (char, ratio), ...]
```

设计理念：
```
图片生成不是直接画整行字，而是要让每个字有自己的位置、偏移、错落关系。所以字符串最终会被转换成“字符 + 排版比例”的列表。
```

具体操作：
```
短标题、黑体、副标题等场景可能直接逐字处理。
部分长度和布局模式下，会给中间某些字增加比例值

折行或错落模式下，也会按标题长度给不同字符预设不同偏移。
```



# 从字符串变为图片，具体步骤是什么？

## 1，初始文字绘制：白色文字 + 深色/黑色背景 

```python
words = get_font_rito(...)
font_image, title_pos = draw_text_normal(...)
```

输入： 
```
words: [(字, 缩放比例), ...]
font_path: 主字体
subtitle_font: 数字/英文备用字体
font_size: 字号
```

输出：
```
font_image: 初始文字图
title_pos: 每个字在图上的 x 区间
```


设计理念：
```
系统不会一开始就画最终效果，而是先画一张“白色文字 + 黑色背景”的临时图。这样后面可以很容易提取文字 mask、切分单字、做轮廓碰撞和纹理贴图。
```

具体操作：
```python
然后逐字绘制：
draw.text((x, y), word, (255, 255, 255), font=font)

同时记录每个字的范围
title_pos = [
    [第1个字起点, 第1个字终点],
    [第2个字起点, 第2个字终点],
]
```

颜色状态：
```
文字颜色：白色 RGB(255, 255, 255)
背景颜色：黑色或接近黑色 RGB(0, 0, 0)
透明度：通常还不是最终透明图
```

## 2, 生成文字 mask：白色文字区域 + 透明背景

输入：font_image: 白字黑底图
输出：masked_image: 白色文字 + 透明背景的 RGBA 图

会遍历像素，
```python
如果像素 RGB 都大于阈值：
    认为是文字区域 → 设为白色不透明
否则：
    认为是背景区域 → 设为透明
```
颜色状态：
```
文字区域：白色，RGBA(255, 255, 255, 255)
背景区域：透明，RGBA(0, 0, 0, 0)
```

## 字符切分：把整行标题切成单字子图

输入：
初始绘制时，系统记录了每个字符的 x 区间：
```
masked_image: 透明底标题图
title_pos: 每个字的 x 坐标区间
mode: 布局模式
font_size: 字号

title_pos = [
    [第一个字起点, 第一个字终点],
    [第二个字起点, 第二个字终点],
    ...
]
```

输出：
```
每个单字子图
每个字的真实轮廓 contour
```

具体操作：
```
先按 title_pos 把整行标题切成单字图：
sub_image = image[:, x_start:x_end]

再提取真实文字区域：
gray = cv2.cvtColor(...)
binary = cv2.threshold(...)
contours = cv2.findContours(...)

得到每个字的轮廓后，再裁剪到文字真实 bounding box。
```


## 动态排版：横排、竖排、错落、折行

调用： ```adjust_layout_original(...)和adjust_layout_mode(...)```

输入：
```
单字透明图列表
单字轮廓列表
标题字符串
布局模式 mode
max_offset: 最大错落幅度
style: 风格
```

输出：
```
重新排版后的完整标题 mask
```

具体操作：
```
系统会为每两个相邻字生成候选偏移：
offset_x: 横向距离
offset_y: 上下错落
offset_style: hori / vert / init

然后把两个字的轮廓画到 mask 上，判断是否相交：
cv2.bitwise_and(mask1, mask2)

如果没有相交，就继续移动，让两个字更靠近；如果相交，就回退一点，留出安全间距。

最后得到每个字的累计偏移：
offset_results = [
    (0, 0),
    (120, -10),
    (230, 20),
]

最后，把所有单字贴回同一张透明画布。
```


颜色状态依然是：
```
文字：白色不透明
背景：透明
```

##  处理第二行标题和副标题
如果有title_second，会单独走一遍：
```
draw_text_normal
segment_wordart
adjust_layout_original

然后和主标题拼接：
font_image = concat_image(font_image, secondtitle_image, hori=False, mid=...)
```

如果有副标题：然后根据 sub_hori 或 sub_vert 排版，再拼接到主标题旁边或下面。


## 合成完整标题 mask

前面每个字已经有自己的偏移量：
```
offsets = [
    (0, 0),
    (120, -10),
    (235, 15),
    ...
]
```

系统会新建一张透明画布，把每个白色单字图按 offset 粘上去。这时得到一张排版后的完整标题图：
```
文字：白色不透明
背景：透明
```

## 准备最终颜色、渐变或纹理图
三种情况：

纯色：color = (255, 255, 255)
渐变：color = [(255, 0, 0), (0, 255, 0), -1]
纹理：系统会从配置里的纹理池随机选一张纹理，比如金属、浅色纹理、暗色纹理。然后```resize/crop/对齐到标题图尺寸```

注意：到这里为止，纯色图、渐变图、纹理图都是“整张铺满”的。

## 合并两张图片
现在有两张图：
```
A：完整标题 mask
   文字白色不透明，背景透明

B：同尺寸颜色/渐变/纹理图
   整张都有颜色

合成：
mask 是文字的地方：保留 B 的颜色/纹理
mask 是背景的地方：变透明

```

## 最终裁切和缩放

输入：
```
refined_img: 贴好纹理的标题图
height: 输出高度
max_width: 最大宽度
```


输出：
```
最终 PNG 文件路径
```

具体操作：
```
先根据 alpha 通道找非透明区域：
alpha > 0 => 计算 bounding box

四周加一点 padding：
上下左右各扩 5 像素

cropped_img = pil_image.crop((x1, y1, x2, y2))

resize_by_height(cropped_img, height, max_width)

```

# 配置信息

项目里的很多效果不是写死在代码里的，而是通过配置文件控制。核心配置文件是：
```
conf/conf.yaml
```

配置主要服务于第二部分“字符串变为图片”，包括字体选择、纹理选择、折行阈值、保存目录和 debug 开关。

## 1，字体配置

配置字段：
```yaml
fonts:
  古装:
  现代:
  悬疑犯罪:
  年代抗战:
  其他:
```

输入：
```
style: 标题风格，比如现代、古装、悬疑犯罪、年代抗战
sent_cut: 是否折行
num: 生成图片数量
fidx: 当前生成第几张
```

输出：
```
font_path: 当前候选图使用的主标题字体
subtitle_font: 数字、英文、副标题使用的辅助字体
```

设计理念：
```
不同风格的标题需要不同字体来表达气质。
古装更适合书法、宋体、毛笔类字体；
现代更适合手写、黑体、创意字体；
悬疑犯罪更适合粗黑、破碎感、压迫感强的字体；
年代抗战更适合厚重、有年代感的字体。
```

具体操作：
```
系统会根据 style 到 fonts 里找到对应字体池。
每个风格下又分成三组字体，num == 3 时，通常每张候选图从不同字体组里选一个字体。

如果标题需要折行，会过滤掉部分不适合长标题或折行场景的字体。
如果 style 找不到，就使用“其他”字体池兜底。
```

## 2，副标题字体配置

配置字段：
```yaml
subtitle_fonts:
  - font/2宋体/思源宋体SC-SemiBold.otf
  - font/1黑体/黑体_思源黑体SourceHanSansCN-Medium.ttf
```

输入：
```
subtitle: 副标题
数字或英文字符
style: 标题风格
```

输出：
```
subtitle_font: 副标题、数字、英文使用的字体
```

设计理念：
```
副标题通常是补充信息，要求清晰、稳定、可读性高。
数字和英文如果直接使用毛笔或手写字体，容易变形或者不协调，所以单独配置一组更规整的字体。
```

具体操作：
```
系统会从 subtitle_fonts 中随机选择一个字体。
如果是年代抗战风格，会固定使用 subtitle_fonts 的第一个字体，让整体更稳定。

在 draw_text_normal 中，如果当前字符是数字或英文，也会改用 subtitle_font 绘制。
```

## 3，纹理配置

配置字段：
```yaml
texture:
  light:
    - [texture/light/new_01.jpg, 1, top]
    - [texture/light/1高清金属拉丝背景.jpg, 0.5, mid]
  dark:
    - [texture/dark/C-6.jpg, 0.5, mid]
```

输入：
```
brightness: light 或 dark
style: 标题风格
color: 用户指定颜色，如果为 None 就使用纹理
```

输出：
```
texture_path: 纹理图片路径
rito: 纹理缩放比例
texture_type: 裁剪方式，top 或 mid
```

设计理念：
```
标题的形状和标题的颜色质感分开处理。
前面先生成白色文字 mask，后面再把纹理贴到文字区域。
这样同一个标题排版，可以快速套不同金属、亮色、暗色或其他材质效果。
```

具体操作：
```
如果 color is None，系统会根据 brightness 从 texture.light 或 texture.dark 中随机选择纹理。
读取纹理后，会按 rito 缩放。
如果纹理尺寸小于标题图，会继续 resize 到能够覆盖标题。

texture_type == mid 时，从纹理中间裁剪；
否则从左上或顶部区域裁剪。

最后用标题 mask 合成：
mask 是文字的地方，保留纹理；
mask 是背景的地方，变成透明。
```

## 4，折行阈值配置

配置字段：
```yaml
folder_pos:
  hori: [5, 5]
  vert: [5, 6]
```

输入：
```
image_vert: 图片是否竖版
style: 标题风格
title_len: 主标题视觉长度
nline: 用户指定单行或双行
```

输出：
```
fold_pos1: 现代风格折行阈值
fold_pos2: 非现代风格折行阈值
sent_cut: 是否需要断句折行
```

设计理念：
```
横版图和竖版图可用空间不同，现代和非现代风格的排版习惯也不同。
所以折行阈值放在配置里，而不是完全写死在代码里。
```

具体操作：
```
如果 image_vert 为 True，就读取 folder_pos.vert；
否则读取 folder_pos.hori。

现代风格主要参考 fold_pos1；
古装、悬疑犯罪、年代抗战、少儿等主要参考 fold_pos2。

当标题视觉长度超过阈值时，sent_cut 会变成 True，后续进入 cut_sentence 做智能断句。
```

## 5，保存目录和 debug 配置

配置字段：
```yaml
debug: False
save_folder: SaveImages
```

输入：
```
style: 标题风格
orig_title: 原始标题
fidx: 当前生成第几张
```

输出：
```
中间图或最终图保存路径
```

设计理念：
```
生成标题图的中间过程比较多，包括原始白字图、透明 mask、排版结果图、贴纹理后的图。
debug 开关用于控制是否保留这些中间结果，方便排查字体、mask、排版和贴图问题。
```

具体操作：
```
最终输出目录一般是：
SaveImages/{style}/{orig_title}/

debug 为 True 时，会保存：
normal_orig.png: 初始白字图
normal_seg.png: 透明 mask
normal_sub.png: 副标题图
normal_temp.png: 拼接后的临时图

debug 为 False 时，服务接口返回 base64 后，会删除部分临时结果，避免磁盘堆积。
```

## 配置整体总结

```
conf/conf.yaml
  ↓
读取字体池、纹理池、折行阈值、保存目录
  ↓
根据 style 选择主字体
  ↓
根据 subtitle_fonts 选择副标题/数字/英文备用字体
  ↓
根据 brightness 选择 light 或 dark 纹理
  ↓
根据 folder_pos 判断是否折行
  ↓
根据 debug 决定是否保留中间图
```

面试里可以这样概括：
```
这个项目把字体、纹理、折行阈值和保存路径都放在配置文件里，代码只负责根据标题风格和生成参数选择合适配置。
这样做的好处是风格扩展比较方便，比如新增一种标题风格时，优先补字体池和纹理池，不需要大改主流程。
```
