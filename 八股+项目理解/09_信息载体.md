# 信息载体
5类：
```text
ThreadState         LangGraph 图状态，会被节点/中间件读写，也可能进入 checkpoint
ThreadData    State 里的一个字段，保存当前 thread 的 workspace/uploads/outputs 真实路径
Runtime       LangGraph 运行时对象，给中间件/工具访问 context/config/state
Config        RunnableConfig，主要放 thread_id、agent_name、model_name 等运行参数
Context       本次 run 的额外上下文，主要放 thread_id


sub-agent 自己的:
SubagentResult    后台任务结果容器
ToolMessage       子 Agent 最终结果返还给父 Agent 的通道
```

对照表
```
ThreadState
保存位置：LangGraph state / checkpoint
生命周期：整个 thread
内容：messages、thread_data、sandbox、artifacts、todos、uploaded_files
用途：Agent 图节点之间传状态

thread_data
保存位置：ThreadState["thread_data"]
生命周期：thread
内容：workspace/uploads/outputs 真实路径
用途：文件工具和 sandbox 路径映射

sandbox
保存位置：ThreadState["sandbox"]
生命周期：thread/run，依 provider 策略
内容：sandbox_id
用途：让工具知道使用哪个沙箱实例

config
保存位置：RunnableConfig，本次 run 参数
生命周期：一次 invoke/stream
内容：thread_id、agent_name、model_name、metadata
用途：控制模型、checkpoint 分区、中间件行为

context
保存位置：Runtime.context
生命周期：一次 run
内容：thread_id 等轻量上下文
用途：中间件/工具快速访问

ModelRequest
保存位置：临时对象
生命周期：一次模型调用
内容：model、messages、system_message、tools、state、runtime
用途：wrap_model_call 中间件修改模型请求

ToolCallRequest
保存位置：临时对象
生命周期：一次工具调用
内容：tool_call、tool、state、runtime
用途：wrap_tool_call 中间件和 ToolNode 执行工具

SubagentResult
保存位置：全局 _background_tasks
生命周期：子任务执行期间
内容：status、result、error、ai_messages
用途：父 task_tool 轮询子 Agent 结果

```


## 1, ThreadState

```python
class ThreadState(AgentState):
    sandbox: NotRequired[SandboxState | None]
    thread_data: NotRequired[ThreadDataState | None]
    title: NotRequired[str | None]
    artifacts: Annotated[list[str], merge_artifacts]
    todos: NotRequired[list | None]
    uploaded_files: NotRequired[list[dict] | None]
```

它继承了 LangChain 的 AgentState, ThreadState 大致保存:
```text
messages        当前会话消息历史：HumanMessage / AIMessage / ToolMessage
sandbox         当前 thread 对应的 sandbox 信息
thread_data     当前 thread 的工作区路径信息
title           会话标题，可选
artifacts       对用户展示的输出文件列表
todos           plan/todo 中间件维护的任务列表
uploaded_files  上传文件元数据
```

## 2, ThreadData 

```python
class ThreadDataState(TypedDict):
    workspace_path: str | None
    uploads_path: str | None
    outputs_path: str | None

保存的是宿主真实路径，例如：
{
  "workspace_path": "C:/.../.deer_flow/threads/bwq-xxxx/user-data/workspace",
  "uploads_path": "C:/.../.deer_flow/threads/bwq-xxxx/user-data/uploads",
  "outputs_path": "C:/.../.deer_flow/threads/bwq-xxxx/user-data/outputs"
}

对应agent能看到的虚拟路径：
/mnt/user-data/workspace
/mnt/user-data/uploads
/mnt/user-data/outputs

```

### ThreadData 如何产生

产生位置在 thread_data_middleware.py (line 1)。
每次 Agent run 开始，before_agent() 会执行：

```python
thread_id = runtime.context.get("thread_id")

如果 context 没有，就从 config 里取：
thread_id = get_config()["configurable"]["thread_id"]

然后根据thread_id计算路径，最后返回：
return {
    "thread_data": {
        **paths,
    }
}

```

LangGraph 会把这个 dict merge 进当前 ThreadState。

注意：这里默认 lazy_init=True，只计算路径，不一定立刻创建目录。真正读写文件时，相关工具/路径逻辑会按需创建。


## 3，SandboxState 保存什么

```python
class SandboxState(TypedDict):
    sandbox_id: str | None
```

它表示当前 thread 绑定的 sandbox 实例 ID。
SandboxMiddleware 在 sandbox/middleware.py (line 1)。如果 lazy_init=False，它会在 before_agent() 里 acquire sandbox：

```python
sandbox_id = provider.acquire(thread_id)
return {"sandbox": {"sandbox_id": sandbox_id}}
```
但是因为是懒更新，只有真正创建sandbox时才会更新

## 4, Config

config 是 LangGraph/LangChain 的 RunnableConfig。它不是对话消息，也不是长期记忆，而是这次 run 的控制参数。

```python
config = {
    "configurable": {
        "thread_id": initial_thread_id,
        "thinking_enabled": False,
        "is_plan_mode": True,
        "model_name": "minimax-m2.5",
        "subagent_enabled": True,
        "tools_enabled": True,
        "agent_name": initial_agent,
    }
}

config["metadata"].update({
    "agent_name": ...,
    "model_name": ...,
    "subagent_enabled": ...,
})
```


```text
thread_id          checkpoint 分区、thread_data 路径、sandbox 绑定
agent_name         加载 agent config、SOUL、agent memory
model_name         当前模型
thinking_enabled   是否开启 thinking
is_plan_mode       是否启用 TodoMiddleware
subagent_enabled   是否暴露 task 工具
metadata           trace / 子 Agent 继承父模型等信息
```

## 5, Context 保存什么

```python
agent.ainvoke(
    state,
    config=config,
    context={"thread_id": current_thread_id}
)

中间件和工具里可以通过 runtime.context 读取。
```

为什么 config 和 context 都放 thread_id？这是冗余但实用的兼容设计：

```text
context.thread_id          给 Runtime / middleware 快速读
config.configurable.thread_id  给 LangGraph checkpoint 和 fallback 用
```

## 6. Runtime 是什么

Runtime 是 LangGraph 在执行中传给 middleware/tool 的运行时对象。你不直接保存它，它是框架生成/传递的临时对象。

通常包含：
```text
runtime.context   本次 run 的 context
runtime.config    RunnableConfig
runtime.state     当前图状态
runtime.store     可选持久 store

当当前runtime是工具运行时，能访问

runtime.state.get("sandbox")
runtime.state.get("thread_data")
runtime.context.get("thread_id")
runtime.config.get("metadata")
runtime.tool_call_id
```

## 7 完整流程

```
state = {"messages": [HumanMessage(content=user_input)]}

agent.ainvoke(state, config=config, context={"thread_id": current_thread_id})


1. LangGraph 根据 thread_id 从 checkpoint 载入旧 ThreadState

2. 把新的 HumanMessage merge 到 state.messages

3. ThreadDataMiddleware.before_agent 计算 thread_data 并写入 state

4. SandboxMiddleware 根据配置懒加载或跳过
5. 其他 before_model / wrap_model_call 中间件处理
6. create_agent 内部构造 ModelRequest
7. DeferredToolFilterMiddleware 可能过滤 request.tools
8. 模型返回 AIMessage
9. 如果 AIMessage 有 tool_calls，进入 tools node
10. ToolNode 构造 ToolCallRequest
11. 工具执行，工具可通过 runtime.state 读取 thread_data/sandbox
12. 工具结果变成 ToolMessage 写回 state.messages
13. 回到模型继续推理
14. 最终 AIMessage 产生
15. after_agent 中间件运行，比如 MemoryMiddleware 入队长期记忆更新
16. checkpointer 保存最终 ThreadState

```

## 8, Checkpoint 里保存哪些 State

```python
create_agent(..., checkpointer=checkpointer, state_schema=ThreadState)

所以 checkpoint 会保存这个thread的图状态

messages
thread_data
sandbox
artifacts
todos
uploaded_files
title
```

注意：runtime.context 不属于 state，不是主要持久化对象。config 也不是业务状态本身，但 checkpointer 会用 configurable.thread_id 做分区键。

## 9，sub-agent 里这些载体怎么传
主 Agent 调用 task 工具时，task_tool 读取父 Agent 的运行态：

```python
sandbox_state = runtime.state.get("sandbox")
thread_data = runtime.state.get("thread_data")
thread_id = runtime.context.get("thread_id")
parent_model = runtime.config.get("metadata", {}).get("model_name")

然后创建：
executor = SubagentExecutor(
    config=subagent_config,
    tools=tools,
    parent_model=parent_model,
    sandbox_state=sandbox_state,
    thread_data=thread_data,
    thread_id=thread_id,
    trace_id=trace_id,
)

```

###  sub-agent 的 State 如何构造

state = {
    "messages": [HumanMessage(content=task)]
}

这表示子 Agent 的对话历史从一条任务 prompt 开始，不继承父 Agent 的完整 messages。
然后：
```python

if self.sandbox_state is not None:
    state["sandbox"] = self.sandbox_state
if self.thread_data is not None:
    state["thread_data"] = self.thread_data

所以子 Agent 的 ThreadState 是：

messages      子 Agent 自己的消息，只包含 delegated task 开始后的内容
thread_data   继承父 Agent 的 thread_data
sandbox       继承父 Agent 的 sandbox 状态，如果有

```

### sub-agent 的 config/context

```
run_config = {
    "recursion_limit": self.config.max_turns,
}

context = {}
if self.thread_id:
    run_config["configurable"] = {"thread_id": self.thread_id}
    context["thread_id"] = self.thread_id
```
所以子 Agent 使用同一个 thread_id。这带来两个效果：
```
文件工具看到同一套 /mnt/user-data 路径
thread_data middleware fallback 时也能算出同一套路径
```
