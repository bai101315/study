# 粘贴过程中怎么确保不会碰撞？

```
我们的标题生成不是直接把整行文字一次性 paste 到最终图上，而是先做字符级排版。第一步用 PIL 绘制白色文字临时图，同时记录每个字符的 x 区间，也就是 title_pos。第二步把图转成透明底 mask，然后根据 title_pos 把整行文字切成单字图。每个单字图会通过 OpenCV 阈值化和 findContours 提取真实文字轮廓。
在布局阶段，系统先根据横排、竖排、错落排版等模式生成一个初始 offset_list。这个 offset 只是候选位置，不直接用于粘贴。然后对每一对相邻字符，把它们的轮廓放到同一个坐标系里，通过 mask 交集判断两个轮廓是否相交。判断方式是把两个 contour 分别画到二值图上，再做 bitwise_and，如果交集有非零像素，就说明发生碰撞。
为了让字符既不重叠也不太松，代码里有一个 move_contours_to_touch 函数。它会沿横向或纵向逐步移动轮廓，直到两个轮廓接近接触，得到一个 move_size。上层再用 move_size 修正当前字符的 offset，同时保留 reserve_size 作为安全间距。最终得到的 offset_results 才会传给 merge_images 做真正的 paste。
所以我们防碰撞的关键不是 paste()，而是 paste 前的“单字切分、轮廓提取、轮廓相交检测、offset 修正”。这种方式比固定字距更稳定，因为它依据的是字体渲染后的真实像素轮廓，能适配不同字体、字号、标点和中英文混排。
```

```
title_pos 解决的是：原始直线排版时怎么切字。
轮廓碰撞解决的是：重新摆放这些字时会不会撞。

后面字符会进行重新摆放：

横向错落：后一个字可能往左压、往上/下偏
竖向布局：后一个字可能放到前一个字下面
folder 布局：可能出现上下两行交错
逗号：会被挪到前一个字左下角
子标题：会压缩字距或居中排列
```

title_pos 是矩形级别的，轮廓检测是像素轮廓级别的。


## 设计原理

```md
项目没有简单按“固定字宽 + 固定字距”排版，因为不同字体的真实字形差异很大。比如毛笔字、手写字、标点、英文数字，它们的视觉轮廓和字体 bbox 往往不一致。只靠 font_size 或字符宽度会出现两个问题：
- bbox 看起来没重叠，但笔画实际挤在一起。
- bbox 间距相同，但视觉上有的很空、有的很挤。
```

## draw_text_normal：先画初始文字，并记录每个字的位置

### 输入：
```
words:
    形如 [(word, scale_rito), ...]
    word 是当前字符 ;
    scale_rito 是缩放比例，比如某些字需要缩小或下移

font_path:
    主标题字体路径

    数字、英文使用的备用字体

    主字体字号
```

### 输出
```

    PIL Image，初始标题图，白色文字画在背景上

x_pos:
    每个字符在 image 中的 x 起止区间
    例如：
    [
        [5, 170],
        [170, 335],
        [335, 500]
    ]
```

### 核心思路
建立一张很大的临时图

逐字绘制
```python

# scale_rito 越大，当前字越小，并且 y 越靠下。

# 逗号，英文特殊处理

# 默认情况下 (x, y) 就是文本块的左上角基准点，text_bbox 会基于这个基准点，计算出文本实际渲染的完整矩形边界
text_bbox = draw.textbbox((x, y), word, font=font)

## segment_wordart：把初始图转成透明底文字 mask

输入
```
输入：白色文字图
输出：透明背景 + 白色文字的 RGBA 图
```

设计原理：
```
因为后面要做轮廓检测。轮廓检测需要知道哪些像素是文字，哪些像素是背景。透明底 mask 能让文字区域非常明确。

文字像素：RGBA(255,255,255,255)
背景像素：RGBA(0,0,0,0)
```

### 输入：
```

    例如 hori_x、vert_x、folder_x、sub_hori、sub_vert

    控制是否启用某些错落布局的参数

### 设计思路
```
1. 把 PIL 图转成 OpenCV 图
2. 按 title_pos 切成单字图
3. 给单字图补边，避免后续移动时没有空间
4. 提取每个字的轮廓 contour
5. 根据布局模式生成 offset，并用轮廓碰撞检测修正 offset
```

contours, hierarchy = cv2.findContours(binary, cv2.RETR_TREE, cv2.CHAIN_APPROX_SIMPLE)
merged_contour = np.concatenate(contours, axis=0)

for idx, (x_start, x_end) in enumerate(title_pos):

## move_contours_to_touch：计算两个字之间能不能贴近，会不会撞

### 输入：
```
image:
    一张空 canvas，只用它的尺寸来创建 mask

contour1_points:
    前一个字符的轮廓点

contour2_points:
    当前字符的轮廓点

offset_style:
    'hori' 或 'vert'
    决定主要沿 x 方向还是 y 方向检测接触距离
```

### 输出
```
return contour1_moved, contour2_moved, (dx, dy)
contour1_moved: 移动后的前一个轮廓
contour2_moved: 当前轮廓
(dx, dy):
    两个轮廓达到接触关系时，前一个轮廓需要移动的距离
    在上层逻辑里，这个值被用来修正当前字符 offset
```

### 核心思路

1. 内部第一步：转凸包。
真实文字轮廓很复杂，直接逐点判断成本更高。凸包可以把字符外形包成一个相对简单的外轮廓，用来估算接触关系。

```
contour1 = cv2.convexHull(contour1_points)
contour2 = cv2.convexHull(contour2_points)
```

2. 取两个轮廓的矩形框。
```
x1, y1, w1, h1 = cv2.boundingRect(contour1)
x2, y2, w2, h2 = cv2.boundingRect(contour2)
```
3. 计算初始移动方向，移动轮廓并判断是否相交

因为这样可以知道两字之间实际还有多少可压缩空间。
如果初始排版太松，就可以压近；如果已经很近，就不会继续压。

```python
contour1_moved = contour1 + np.array([[[dx, dy]]])
intersect = contours_intersect(image, contour1_moved, contour2_moved)

while not intersect:
    if offset_style == 'vert':
        dy += 1
    else:
        dx += 1

    contour1_moved = contour1 + np.array([[[dx, dy]]])
    intersect = contours_intersect(image, contour1_moved, contour2_moved)
```   

4. contours_intersect：判断两个轮廓有没有碰撞
   
输入
```
image: 用来确定 mask 大小
contour1: 第一个字符轮廓
contour2: 第二个字符轮廓
```

核心代码：
1. 创建两张黑色 mask。
2. 把 contour1 涂成白色。
3. 把 contour2 涂成白色。
4. 两张 mask 做 bitwise_and。
5. 如果交集区域存在白色像素，说明两个轮廓重叠。
```
img1 = np.zeros((image.shape[0] + 100, image.shape[1]), dtype=np.uint8)
img2 = np.zeros((image.shape[0] + 100, image.shape[1]), dtype=np.uint8)

cv2.drawContours(img1, [contour1], 0, 255, -1)
cv2.drawContours(img2, [contour2], 0, 255, -1)

img_and = cv2.bitwise_and(img1, img2)
_, intersect = cv2.threshold(img_and, 127, 255, cv2.THRESH_BINARY)
non_zero_pixels = cv2.countNonZero(intersect)

return non_zero_pixels > 0
```

## merge_images：最终粘贴

## 例子 - 亲爱
```
title_pos = [
    [5, 170],     # 亲
    [170, 335]    # 爱
]

1. 用 [5,170] 切出“亲”的图
2. 用 [170,335] 切出“爱”的图
3. 分别提取“亲”和“爱”的轮廓
4. 初始认为“爱”放在“亲”的右边
5. 检测两个字的真实轮廓距离
6. 如果中间太空，就把“爱”往左压
7. 但保留 5px 或 reserve_size
8. 得到最终 offset
9. paste 到最终透明画布
```
