# with ... as
with ... as 是 Python 中上下文管理器（Context Manager）的语法糖

```python
with 上下文表达式 [as 变量名]:
    # 代码块（上下文内的逻辑）

# 自动打开、自动关闭文件，无需手动调用 f.close()
with open("test.txt", "r", encoding="utf-8") as f:
    content = f.read()
# 跳出 with 块后，文件已自动关闭
```
上下文表达式：执行后返回一个上下文管理器对象; 


完整执行步骤：
```
执行 with 后的表达式，得到上下文管理器对象
调用上下文管理器的 __enter__() 方法，执行进入上下文的初始化逻辑
如果写了 as 子句，将 __enter__() 的返回值赋值给 as 后的变量
执行 with 缩进内的代码块
无论代码块正常执行完毕，还是抛出异常中断，都会调用上下文管理器的 __exit__() 方法，执行资源清理、现场还原等逻辑
```

## 上下文管理器
一个类只要实现了以下两个魔法方法，就符合上下文管理器协议

```
__enter__(self):
作用：进入 with 块前自动调用，负责资源初始化、连接建立等逻辑
返回值： 返回值会赋值给 as 后的变量（无 as 则忽略返回值）

__exit__(self, exc_type, exc_val, exc_tb)
作用： 退出 with 块时自动调用，负责资源释放、连接关闭、异常处理等逻辑，无论代码是否抛异常都会执行
返回值：返回布尔值，控制异常是否向外传播
```
## __exit__ 的三个异常参数：
正常结束时：三个参数均为 None

当代码块抛出异常时
```
exc_type：异常的类型（如 ValueError、TypeError）
exc_val：异常的实例对象，包含异常的具体信息
exc_tb：异常的追踪栈（traceback）对象
```

常见考点：
```
__exit__: True, 异常会被抑制，程序会继续执行 with 块之后的代码
__exit__：False, 继续向外抛出
```
