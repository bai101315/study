# 项目中的记忆机制有没有遇到什么问题？是如何解决的？

```md
这个记忆机制不是我从零设计出来的，我主要是基于已有项目做了阅读、复现和理解。所以如果问“我在实现过程中遇到了什么问题”，我不能说我亲自踩过很多坑。更准确地说，我在理解这个模块时发现它要解决几个典型问题：session 状态和长期记忆的职责边界、同步更新 memory 带来的延迟、以及 memory 注入 prompt 时的 token 控制。

我做的工作主要是把这些机制读清楚，并能解释为什么它这样设计。例如 Checkpointer 保存当前会话状态，memory.json 保存长期事实；MemoryMiddleware 会过滤工具消息和上传文件块；记忆更新通过 queue 和 debounce 异步处理；注入 prompt 时有 token budget，避免把整个 memory 文件都塞进上下文。

所以我不会把这部分包装成“我遇到了很多线上问题然后优化”，而会说这是我通过阅读和复现项目理解到的工程设计点。如果要讲个人贡献，我会更聚焦在我真正改过或验证过的地方，比如 tool_search 延迟工具发现、token benchmark、MCP 工具 schema 延迟暴露这些。
```
